【STEP 1 — 系統介紹（口頭說明，30 秒）】

  說：「這是一個智慧植物監控系統，結合了
       ESP32 邊緣硬體、Go 後端 API、Python 機器學習
       還有 Web 儀表板，可以全天候監控植物生長狀態。」

  指向硬體：
  - ESP32 板子 + OLED 螢幕（展示 RoboEyes 動畫）
  - DHT11 溫濕度感測器
  - 土壤濕度感測棒插在土裡
  - BH1750 光照感測器
  - 沈水馬達 + 繼電器

================================================================

【STEP 2 — 登入系統】

  1. 開啟瀏覽器，輸入：http://<Raspberry Pi IP>:3000/frontend/login.html
  2. 輸入帳號密碼，點擊登入
  3. 系統自動跳轉到 Dashboard

================================================================

【STEP 3 — 展示即時感測數值】

  指向儀表板上方五張 stat card：
  - 溫度（°C）
  - 濕度（%）
  - 土壤濕度（%）
  - 光照強度（lux）
  - 植物心情（emoji）

  說：「這些數值是 ESP32 每 5 分鐘透過 MQTT 傳上來的，
       Python 機器學習模型會分析這些數據，
       預測出植物目前的『心情』狀態。」

================================================================

【STEP 4 — 展示 OLED 表情動畫】

  指向 ESP32 的 OLED 螢幕：
  說：「心情不只顯示在網頁上，
       同時也會透過 MQTT 傳回給 ESP32，
       用 RoboEyes 函式庫在 OLED 上顯示對應的表情動畫。」

  （若心情是 happy → 笑眼動畫；sad → 向下眼神；hot → 憤怒眼神）

【STEP 6 — 展示灌溉控制（水泵）】

  1. 找到「灌溉控制」那張卡片
  2. 確認狀態是「馬達待機」（灰色點）
  3. 點擊「💧 灌溉」按鈕
  4. 指向馬達，讓觀眾看到水馬上開始流
  5. 說：「透過 MQTT 指令，可以遠端控制沈水馬達澆水，
         1 秒後會自動關閉，也可以手動按停止。」
  6. 點擊「停止」確認可以手動關閉

【備用 QA 常見問題】

Q：為什麼不直接用 HTTP 控制 ESP32？
A：ESP32 是 HTTP Client，沒有 HTTP Server。
   MQTT 是 IoT 標準的雙向通訊協定，
   Broker 由 Raspberry Pi 負責，更適合這個架構。

Q：機器學習模型怎麼訓練的？
A：用模擬感測資料集訓練 Random Forest 分類模型，
   輸入四個特徵（溫度、濕度、土壤、光照），
   輸出 happy / sad / sick / hot / normal 五種心情。

Q：這個系統可以擴展多台植物嗎？
A：可以。每台 ESP32 有獨立的 DEVICE_ID，
   後端 API 和照片存儲都已經按裝置 ID 分類。
